================================================================================
LESSONS LEARNED
================================================================================

1. ALWAYS TEST BEFORE PRODUCTION
   - Issue: First email had incorrect filter (missing status check)
   - Solution: Create test prospect, verify email format, check Airtable updates
   - Protocol: Test → Verify → Deploy

2. EMAIL FORMATTING IN GMAIL
   - Issue: HTML   
 tags rendered as plain text
   - Solution: Use separate <p> tags with margin: 0; padding: 0;
   - Best Practice: Wrap signature in individual <p> tags

3. SAFEGUARDS AGAINST MASS MAILINGS
   - MAX_EMAILS_PER_RUN = 1 (only 1 email per execution)
   - DRY_RUN mode (test without sending)
   - Email validation (regex check)
   - State tracking (SQLite database)

4. AIRTABLE FILTER FORMULA SYNTAX
   - Use: AND(NOT({Email} = BLANK()), {In Automation} = TRUE(), {Qualification Status} = 'New')
   - All three conditions must be TRUE
   - Field names with special characters need proper escaping

5. LAUNCHAGENT VS CRON
   - LaunchAgent: macOS native (preferred for Mac)
   - StartInterval: 900 seconds = 15 minutes
   - RunAtLoad: true = starts automatically on login
   - Logs: StandardOutPath and StandardErrorPath capture all output

TROUBLESHOOTING
================================================================================

Script not running?
  1. Check status: launchctl list | grep com.meraglim.script_02_qualification_email
  2. Check errors: tail -f ~/Automations/logs/script_02_launchagent_error.log
  3. Restart: launchctl unload ... && launchctl load ...

No emails being sent?
  1. Check filter: Email + In Automation + Status = "New"
  2. Run manual test: /usr/bin/python3 ~/Automations/scripts/script_02_airtable_qualification_email_oauth.py
  3. Check logs: tail -f ~/Automations/logs/script_02_airtable_qualification_email.log

Duplicate emails sent?
  1. Clear state: rm ~/Automations/config/state.db
  2. Restart: launchctl unload ... && launchctl load ...

Gmail OAuth error?
  1. Delete token: rm ~/Automations/config/oauth_token.json
  2. Run script manually to re-authenticate
  3. Follow browser login flow
